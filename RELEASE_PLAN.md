**План (2 уровня — выбрать в вопросах):**
- **A. Быстро:** убрать plaintext, хранить только SHA-256 хеш; сверять при входе; почистить `shared/admins` в Firebase; пароль убрать из истории git (BFG/filter-repo).
- **B. Правильно:** перевести админ-доступ на Firebase Auth + custom claims + `database.rules.json` (только админ пишет в `shared/*`).
**Риск:** пароль уже в истории git — нужна чистка истории + смена пароля.

## 13. Аудит данных и лимитов (Firebase/Supabase/git)
**Находки:**
- **git раздут:** `.git` 817 МБ, MP3 28–86 МБ — близко к лимитам GitHub (100 МБ/файл, репозиторий желательно <1 ГБ). **План:** вынести аудио в Git LFS или внешний CDN/Storage; почистить историю.
- **Firebase Spark (free):** RTDB 1 ГБ хранилище, ~**100 одновременных подключений**, 10 ГБ/мес выгрузка → потолок ~100 онлайн одновременно. Профили лёгкие — хранилища хватит надолго.
- **Supabase free:** 500 МБ БД, 50k MAU — пока почти не используется.
- **AI/TTS free:** зависит от выбранного провайдера (лимиты запросов).
**Вывод:** для старта (десятки-сотни учеников) хватит; узкое место — 100 одновременных коннектов Firebase и размер git.
