# Нативный корейский голос — TTS-прокси 🌸

Маленький Cloudflare Worker, который превращает корейский текст в естественную речь
через **Google Cloud Text-to-Speech** (голоса `ko-KR-Neural2`). Приложение обращается
к нему за озвучкой; если воркер не настроен — всё работает как раньше, на встроенном
голосе браузера (Web Speech API).

> 🔒 **Безопасность.** Ключ Google живёт ТОЛЬКО в секретах воркера на сервере.
> В коде сайта (`app.js`) ключей нет и быть не должно.

---

## Зачем

Встроенный голос браузера на части телефонов звучит с акцентом или вовсе
отсутствует. Этот прокси даёт один и тот же приятный женский голос всем ученицам,
независимо от устройства. Результаты кешируются — повторные слова не тратят квоту.

---

## Что понадобится

1. Аккаунт **Cloudflare** (бесплатный тариф: 100 000 запросов в день).
2. Аккаунт **Google Cloud** с включённым **Text-to-Speech API** и API-ключом
   (бесплатно до 1 млн символов Neural2 в месяц).
3. **Node.js** на компьютере (для команды `wrangler`).

---

## Шаги деплоя

### 1. Получить ключ Google Cloud TTS
1. Зайди на https://console.cloud.google.com/ → создай проект (или возьми существующий).
2. **APIs & Services → Library** → найди **Cloud Text-to-Speech API** → **Enable**.
3. **APIs & Services → Credentials → Create credentials → API key** → скопируй ключ.
4. (Рекомендуется) нажми на ключ → **API restrictions** → ограничь его только
   *Cloud Text-to-Speech API*, чтобы ключ нельзя было использовать ни для чего другого.
5. (Рекомендуется) в **Billing → Budgets & alerts** поставь бюджет-лимит, чтобы
   не уйти за бесплатный объём незаметно.

### 2. Установить инструмент Cloudflare
```bash
npm install -g wrangler
wrangler login
```

### 3. Залить воркер
Из этой папки (`serverless/tts-proxy`):
```bash
# положить ключ Google в секрет (он не попадёт в код и в git)
wrangler secret put GOOGLE_TTS_KEY
# вставь ключ, когда попросит, и нажми Enter

# (необязательно) ограничить CORS своим доменом вместо "*"
#   отредактируй ALLOWED_ORIGIN в wrangler.toml

# опубликовать
wrangler deploy
```
После деплоя wrangler покажет адрес вида:
```
https://km-tts.<твой-поддомен>.workers.dev
```

### 4. Подключить в приложении
1. Открой приложение → **Профиль → админка (Мади) → Школа → блок «Озвучка»**.
2. Вставь адрес воркера в поле и нажми **Сохранить**.
3. Готово. Адрес сохранится в Firebase и автоматически включит естественный голос
   у всех учениц. Чтобы вернуться на голос браузера — очисти поле и сохрани.

---

## Проверка

Открой в браузере (подставь свой адрес):
```
https://km-tts.<поддомен>.workers.dev/tts?text=안녕하세요&voice=ko-KR-Neural2-A
```
Должен скачаться/проиграться MP3 с корейским «Аннёхасэё».

---

## Параметры эндпоинта `/tts`

| Параметр | Значение | По умолчанию |
|----------|----------|--------------|
| `text`   | корейский текст (до 300 символов) | — (обязательно) |
| `rate`   | скорость 0.25–4.0 | `1.0` (медленный режим — `0.7`) |
| `voice`  | `ko-KR-Neural2-A/B/C`, Wavenet, Standard | `ko-KR-Neural2-A` |

Приложение само подставляет женский голос (`-A`) для слов и Мади, мужской (`-C`) —
для мужских реплик в диалогах и аудио-скриптах ТОПИК.
