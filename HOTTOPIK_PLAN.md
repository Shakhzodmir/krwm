Карта 제1회 (по обложке + содержанию 해설집, требует уточнения точных границ на шаге 1):

- `Hot_topik2...pdf` (해설집), по 목차: 제1회 **정답** стр. 54, **해설** стр. 55 → до стр. 97 (дальше начинается 제2회, стр. 98).

## 0.1 ⚠️ Вопрос про копирайт

`NEW_MATERIALS_PLAN.md` (п.0.3) уже фиксирует принцип для всей папки `assets/new material/`: учебники сторонних издателей — это **референс**, а не контент для дословной публикации; страницы/тексты/картинки 1:1 в приложение не идут, PDF в git не попадают (папка в `.gitignore`).

Hot TOPIK — коммерческая книга **издательства 한글파크** (а не официальный архив topik.go.kr, как уже добавленные реальные 회차 41/47/52/60/64/83/91/96/102). Формально это ближе к NIIED-учебникам, чем к официальным прошедшим экзаменам.

**Решение пользователя:** переносим дословно, как в книге (вопросы/диалоги/тексты/варианты — без перефразирования). Картинки-сценки 듣기 1-3 — кроп напрямую из PDF (см. п.1.1). Риск по авторскому праву подтверждён и принят осознанно — материал для собственного учебного приложения, не для дальнейшей публикации/продажи самой книги.

## 1.2 Инструменты (только в scratchpad, проект не трогают)

В системе нет `pdftoppm`/poppler → штатный Read-PDF тула не работает на сканах. Поднял локально (не в проекте, не в git):
- `pdfjs-dist` (уже стоит глобально) — рендер страницы PDF.
- `@napi-rs/canvas` (поставил в scratchpad) — растеризация в PNG + кроп нужной области.

Скрипты лежат в `…\scratchpad\pdfrender.mjs` — рендерит диапазон страниц в PNG с заданным масштабом; кроп делаю тем же канвасом по пикс. координатам.

## 2. Шаги (пилот — только 제1회)

1. **Границы страниц** — отрендерить и пробежать `hot topik.pdf` стр. 1–50, найти: конец 듣기/начало 쓰기, конец 쓰기/начало 2교시(읽기 обложка+инструкция), конец 읽기/начало 제2회. Аналогично в 해설집 стр. 54–97 — где там 듣기-разбор / 쓰기-разбор / 읽기-разбор, есть ли расшифровка скриптов 듣기.
2. **듣기 (50 вопросов)** — по страницам-сканам выписать диалоги (своими словами, см. п.0.1), варианты, сверить правильный ответ с таблицей 정답 (стр. 54), написать русский разбор. Кропнуть картинки для вопросов 1–3 (и любых других картиночных, если найдутся дальше).
3. **쓰기 (4 задания)** — 51/52 (вставить предложение в ㉠/㉡) и 53/54 (мини-эссе) с `model` и `check`, как в существующих реальных экзаменах. Кропнуть график для №53.
4. **읽기 (50 вопросов)** — passage/varianты/ответ (сверка с 정답), разбор на русском.
5. **Собрать объект** `8001` и добавить в `TOPIK2_EXAMS`.
6. **Проверить UI** — карточка появляется в группе «Пробные и другие» на экране TOPIK II, тест открывается, 듣기 озвучивается (TTS + смена тона 남/여), картинки показываются, 읽기 проверяется автоматически, 쓰기 — экран самооценки.
7. **Ручная проверка в браузере** — пройти по несколько вопросов каждой части, тёмная тема + 375px/768px (стандарт проекта).
8. После ОК от тебя — повторить шаги 1–7 для 제2~5회 отдельными заходами.

## 3. Риски

- Объём — 104 вопроса + 4 письменных задания на один экзамен; точность сверяю по таблице ответов на каждом шаге, чтобы не получить тест с неправильным ключом.
- Точные пиксельные границы кропа картинок — подбираются вручную по каждой странице.
- Если на шаге 1 окажется, что 5 экзаменов раскладываются по страницам не равными блоками (~40 стр/экзамен) — границы уточню по факту, а не по оценке.
